# Evaluation Framework

## Purpose
Create a repeatable method for judging whether a prompt is actually good.

## Core Criteria
1. Clarity
2. Relevance
3. Structure
4. Consistency
5. Practical usefulness
6. Adaptability
7. Hallucination resistance
8. Editability

## Scoring Method
Score each criterion from 1 to 5.

## Questions To Ask
- Did the output follow instructions?
- Was the result useful without heavy cleanup?
- Did the model invent details?
- Was the structure easy to work with?
- Would this perform consistently with similar inputs?

## Revision Loop
1. Run the prompt
2. Review the output
3. Identify weak spots
4. Adjust context, constraints, or format
5. Re-test
6. Compare results

## Takeaway
Prompt quality should be judged by performance, not vibes.
